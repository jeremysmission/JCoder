## R13: Open Knowledge Fallback + Retrieval Tiering
**Priority:** HIGH -- censorship experiment proved JCoder refuses all non-indexed questions
**Goal:** Let JCoder answer general knowledge questions without source material

### Tasks
- R13.1: Add open_knowledge_fallback mode to agent/bridge.py
  - When retrieval returns 0 results OR all scores < 0.15, skip RAG context
  - Let the LLM answer from parametric knowledge with a disclaimer
  - Config toggle: `agent.yaml -> open_knowledge_fallback: true`
- R13.2: Quality-tiered retrieval in federated_search.py
  - Hot tier: quality_score >= 3 (search first)
  - Cold tier: quality_score < 3 (fallback if hot returns < 3 results)
  - Measurable via telemetry: tier_used field
- R13.3: Tests (target: 15+)

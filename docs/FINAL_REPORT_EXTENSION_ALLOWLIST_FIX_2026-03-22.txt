===============================================================================
                        VULNERABILITY ANALYSIS
===============================================================================

VULNERABLE FILES FOUND (4):
1. tools/site_wiki_builder.py
   - Issue: Hardcoded DOC_TYPES with .xls, .ppt, .odt, .ods, .odp
   - Risk: Tools wouldn't auto-discover these formats if core adds support

2. tools/wiki_builder.py
   - Issue: Hardcoded TEXT_EXTENSIONS with .rst, .svg
   - Risk: Stale list if core updates format support

3. tools/wiki_builder_v2.py
   - Issue: Hardcoded TEXT_EXTENSIONS with .rst, .svg
   - Risk: Duplicate hardcoding (copy from wiki_builder.py)

4. tools/fts5_demo.py
   - Issue: Hardcoded INDEX_EXTENSIONS with .rst
   - Risk: Not synced with core LANGUAGE_MAP

ROOT CAUSE:
- Tools had independent extension lists (not importing from core)
- Pattern matches HybridRAG3_Educational vulnerability
- Silent divergence would occur if core updated

===============================================================================
                            THE FIX APPLIED
===============================================================================

PATTERN: Single Source of Truth + Runtime Imports

FROM (Vulnerable):
    TEXT_EXTENSIONS = {".py", ".js", ".rst"}  # Hardcoded, will drift

TO (Safe):
    from ingestion.chunker import LANGUAGE_MAP

    TEXT_EXTENSIONS = {".py", ".js"}  # Base set
    if _HAS_LANGUAGE_MAP:
        TEXT_EXTENSIONS.update(LANGUAGE_MAP.keys())  # Dynamic, always synced

BENEFITS:
✓ Single source of truth (LANGUAGE_MAP in chunker.py)
✓ Runtime imports (no copying = no drift)
✓ Graceful degradation (works without ingestion module)
✓ Auto-discovery (new formats automatically included)
✓ Future-proof (no manual sync needed when core updates)
